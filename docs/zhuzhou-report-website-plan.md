# 抓周 MBTI 報告網站設計方案

## 1. 產品定位

此網站提供家長在抓周儀式後，快速輸入三輪道具結果，系統自動推導寶寶版 MBTI 並生成可分享、可下載、可商業化的個人化報告。

### 目標使用者
- 0–3 歲寶寶家長
- 抓周活動主辦單位（攝影、主持、場地方）
- 親子品牌（聯名報告封面）

### 核心價值
- **儀式感數位化**：把抓周儀式轉成可保存的成長紀錄
- **即時可解讀**：三輪結果即時輸出性格摘要與教養建議
- **可商業擴充**：付費、品牌客製、社群分享、名單蒐集

---

## 2. 使用者流程（前台）

## 2.1 首頁
**目的**：教育與轉換

**內容模組**：
1. Hero 區塊：一句話說明「三輪抓周 → 寶寶版 MBTI 報告」
2. 抓周理念：抓周不是命定，而是偏好線索
3. 流程導覽：三輪選道具 → 計分 → 報告
4. 範例報告預覽：展示 1–2 份模擬報告
5. CTA：立即開始

## 2.2 抓周選擇頁（三輪）
**步驟**：
1. 第一輪：選「興趣（感知世界方式）」道具
2. 第二輪：選「能力（處理問題方式）」道具
3. 第三輪：選「人格特質（成為怎樣的人）」道具

**UX 建議**：
- 每輪顯示進度條（Round 1/3, 2/3, 3/3）
- 道具卡顯示：名稱、寓意、簡短關鍵詞
- 防呆：每輪必選 1 項，選完可返回修改

## 2.3 結果頁
輸出內容：
- MBTI 類型（例如 ENFP）
- 個性摘要（寶寶語氣）
- 教養建議（家庭互動、學習情境）
- 適合發展方向（遊戲、活動、環境）

## 2.4 下載頁
支援：
- PDF 匯出（A4 / 社群直式）
- 圖片匯出（1080x1920 或 1080x1350）
- 分享按鈕：LINE / IG（下載後引導分享）

---

## 3. 後台資料與邏輯設計

## 3.1 道具資料表（props）
欄位建議：
- `id`：UUID
- `round_type`：enum(`interest`, `ability`, `personality`)
- `name_zh`：道具名稱
- `meaning`：寓意
- `mbti_axis`：主軸（E/I、S/N、T/F、J/P）
- `axis_value`：對應值（例如 E）
- `weight`：該輪權重（預設 1）
- `active`：是否上架
- `sort_order`：排序

## 3.2 MBTI 對照表（mbti_profiles）
- `mbti_type`：16 型之一
- `title`：類型標題
- `summary`：個性摘要
- `parenting_tips`：教養建議（JSON）
- `development_paths`：發展方向（JSON）
- `cover_theme`：預設封面主題

## 3.3 報告模板（report_templates）
- 多版型（品牌版 / 通用版）
- 可替換欄位：品牌名、寶寶名、日期、照片、MBTI 文案
- 支援 A4 與手機長圖尺寸

---

## 4. MBTI 計分引擎（建議規則）

### 輸入
三輪各一個道具，每個道具可對 1 個或多個 MBTI 軸加分。

### 計分方式
1. 對四個維度建立分數桶：`E/I`、`S/N`、`T/F`、`J/P`
2. 將每輪道具的 `axis_value` 加入對應桶分數
3. 每輪可設定權重（例：興趣 1、能力 1.2、人格 1.3）
4. 每個維度取分數高的一側，組成 4 碼 MBTI

### 同分處理
- 優先採用第三輪（人格）作為 tie-breaker
- 若仍同分，採用第二輪（能力）
- 最後仍同分，回傳「混合型」並提示「可再觀察」

---

## 5. 商業化功能規劃

## 5.1 付費生成報告
- 前 1 頁免費預覽，完整報告需付款解鎖
- 金流：Stripe / 綠界 / 藍新（依市場）
- 訂單狀態：待付款、已付款、退款

## 5.2 留 email 領取
- 未付款可先留 email 取得精簡版
- 已付款自動寄完整版 PDF 下載連結
- 建議串接 EDM 系統（如 Mailchimp / Brevo）

## 5.3 客製封面與照片
- 上傳寶寶照片（限制大小與格式）
- 選擇主題封面（生日、節慶、品牌聯名）
- 生成時加上浮水印與報告編號

## 5.4 社群分享
- 生成分享圖卡（含 MBTI 與一句文案）
- LINE 一鍵分享連結
- IG 下載後引導（Web 限制通常無法直接貼文）

## 6. 技術架構建議（MVP）

### 前端
- Next.js（App Router）
- Tailwind CSS
- i18n（繁中優先）

### 後端
- Next.js API Route / NestJS
- PostgreSQL
- Prisma ORM

### 產生報告
- HTML Template + Puppeteer（輸出 PDF）
- 圖片可用 `satori` / `@vercel/og` 產生 OG 風格圖卡

### 儲存與媒體
- S3 相容儲存（AWS S3 / Cloudflare R2）
- CDN 提升報告下載速度

### 分析與追蹤
- GA4 / PostHog
- 漏斗：首頁 → 完成三輪 → 付款 → 下載

---

## 7. MVP 範圍與時程

## Phase 1（2–3 週）
- 首頁、三輪選擇、結果頁
- 基礎 MBTI 計分
- 基本 PDF 匯出

## Phase 2（2 週）
- 付款、email、品牌客製
- 圖片分享卡、下載頁優化

## Phase 3（持續）
- A/B 測試文案
- 更多模板與聯名封面
- 活動主辦端儀表板（批次建立報告）

---

## 8. 風險與合規提示

- **醫療/心理聲明**：需註明非醫療或人格診斷工具
- **兒童個資**：照片、姓名、生日屬敏感資料，需同意與刪除機制
- **報告定位**：建議文字偏「成長觀察建議」，避免定型化標籤

---

## 9. 你現在就可以落地的下一步

1. 先建立 `props` 與 `mbti_profiles` 兩張表
2. 把你提供的 Notion 道具資料整理成 CSV 匯入
3. 實作三輪選擇頁 + 計分 API
4. 先產出文字版結果頁，確認文案後再做 PDF 模板
5. 最後接金流與 email 自動寄送
